What the case is designed to test

Four things any model has to get right

Discordance
MORPH ↔ FLOW
Morphology says 18% blasts. Flow says 22%.
The model must reconcile — not silently pick one number.
Hedge resolution
MORPH ↔ FLOW
Morphology hedges on lineage. Flow proves monocytic differentiation.
The model must credit flow with resolving the hedge.
Single-source classifying
MOLEC only
NPM1 + FLT3-ITD appear only in molecular.
Karyotype and FISH are normal. The whole diagnosis hinges here.
Lane discipline · the trap
DNMT3A R882H
Real and Tier II — but it does NOT classify the disease.
Belongs in prognostic notes — not in the diagnosis line.

The architectural pattern

Two stages, one pattern.

Stage 1
PDF Intake
JSON
Reads all four PDFs together. Emits a structured JSON with per-source findings, concordances, discordances, single-source findings, and a classifying boolean on every variant. Every finding carries a verbatim phrase copied from the source.
structured JSON
no raw PDFs pass
Stage 2
WHO Classifier
JSON
11 sections + trace
Receives only the structured JSON — not the raw PDFs. Its prompt enforces explicit rules. Output: the 11-section integrated report plus a per-sentence evidence trace mapping each sentence to its source.
Two LLM calls. Each with one specific job. Each with one editable system prompt.

Component 2 of 7 · Stage 1 · main reasoner

PDF Intake — read four PDFs together, emit structured findings

multimodal LLM · vision-capable · strict JSON out
reads all four PDFs in one LLM call and emits cross-PDF findings.
  • • Multimodal. Both extracted text and embedded image bytes go to the vision-capable model.
  • • Per-source tagging. Every finding carries a source_id + a verbatim phrase copied from the source.
  • • Concordances, discordances, single-source findings broken out as named fields, not free prose.
  • • classifying boolean on every variant. Disease-defining vs prognostic decided at extraction time.
  • • Output is strict JSON. Same shape every run.
The larger of the two main reasoners. Its system prompt is THE main editable lever — change how this reads, and everything downstream sees a different world.

✶ WHO Classifier · System Prompt
excerpt · what its prompt encodes
1›Use only what's in the Stage 1 JSON. No outside knowledge.
2›Resolve every discordance out loud — no silent picks.
3›Name every single-source finding. Be explicit.
4›Non-classifying variants stay in prognostic notes, never the diagnosis line.
5›Output: 11 fixed sections + Part B per-sentence trace.

    Wire it up — four connections
    Chat Input → Agent

    Agent → Chat Output
